# Web Radar Collector 使用说明

这是一款用于人工浏览采集的网页采集插件。它适合在账号、接口或自动化采集不可用时，手动收集网页里的图片、视频帧和文本，并在结束任务后导出一个 Markdown 表格和本地截图素材。

## 1. 安装插件

1. 打开 Chrome 地址栏输入 `chrome://extensions/`。
2. 打开右上角「开发者模式」。
3. 点击「加载已解压的扩展程序」。
4. 选择解压后的插件目录，也就是包含 `manifest.json` 的那个文件夹。
5. 安装后建议把扩展固定到浏览器工具栏，方便查看采集状态。

## 2. 第一次采集会先新建任务

进入需要采集的网页后，把鼠标移动到图片或视频区域，右上角会出现「采集」按钮。

如果当前还没有进行中的任务，点击「采集」后会弹出「新建采集任务」窗口。输入任务名后，点击「新建并采集」，插件会先创建任务，再把当前内容加入这次任务。

任务名建议写成这次采集的主题，例如「心声」「竞品评论观察」「网页功能反馈」。

## 3. 采集文本

选中页面里需要保存的文字，松开鼠标后，选区旁边会出现「采集」按钮。点击后，插件会保存这段文字和当前页面链接。

文本采集适合保存评论、标题、正文片段、用户反馈、功能描述等内容。

## 4. 采集图片和视频帧

鼠标移动到图片上时，点击「采集」会保存当前图片相关信息。

鼠标移动到视频上时，点击「采集」会截取当前视频画面作为视频帧记录。视频采集前可以先暂停到想保存的画面，再点击「采集」。

采集按钮会尽量贴近当前图片或视频区域，鼠标在媒体附近移动时按钮不会立刻消失。

## 5. 查看当前任务状态

点击浏览器工具栏里的扩展图标，可以看到当前任务名称、已经采集的记录数量、空闲时间，以及最近采集的记录。

如果 20 分钟没有新增采集，插件会自动结束任务并导出。也可以手动点击「结束并导出」。

## 6. 导出结果

点击「结束并导出」后，Chrome 会在默认下载目录下创建一个任务文件夹，并下载本次任务的 Markdown 文件和素材文件。

任务文件夹名格式为：

```text
yyyy-mm-dd hh：mm：ss 任务名
```

文件夹里通常包含：

- `任务名.md`：本次任务的 Markdown 表格记录。
- `assets/`：采集到的图片和视频暂停帧截图。

导出的 Markdown 表格格式为：

```markdown
| 编号 | 页面 | 采集内容 |
|---:|---|---|
| 1 | [页面标题](页面链接) | 采集到的文本<br>![图片](assets/001-xxx.png) |
```

说明：

- 编号按采集顺序正序排列。
- 页面列是可点击的 Markdown 超链接，显示文字最多 10 个字，链接本身保持完整。
- 采集内容列可以包含选中文本、图片说明、视频帧说明，以及本地截图。
- 图片和视频帧会同时保存到 `assets/` 文件夹，Markdown 中的图片会引用这些本地文件。

## 7. 常见问题

### 看不到「采集」按钮

刷新当前页面，确认扩展已经在 `chrome://extensions/` 中启用。如果刚刚修改或更新过插件，需要在扩展管理页点击「重新加载」。

### 点击采集没有弹出新建任务窗口

请先重新加载插件，再刷新页面。正常情况下，没有进行中的任务时，点击页面上的「采集」会直接弹出「新建采集任务」窗口。

### 导出文件在哪里

文件会下载到 Chrome 默认下载目录。文件夹名包含导出时间和任务名，例如：

```text
2026-04-29 12：30：00 心声
```

Windows 路径不能使用英文冒号，所以文件夹名里的时间使用全角冒号 `：`。

### Markdown 里图片显示不出来

请保持 `任务名.md` 和 `assets/` 文件夹在同一个任务文件夹里，不要只单独移动 Markdown 文件。Markdown 里的图片路径是相对路径，移动文件后可能会找不到截图。
